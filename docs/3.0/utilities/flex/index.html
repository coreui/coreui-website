Flex

Quickly manage the layout, alignment, and sizing of grid columns, navigation, components, and more with a full suite of responsive flexbox utilities. For more complex implementations, custom CSS may be necessary.

Enable flex behaviors

Apply display utilities to create a flexbox container and transform direct children elements into flex items. Flex containers and items are able to be modified further with additional flex properties.

I'm a flexbox container!
<div class="c-d-flex c-p-2 bd-highlight">I'm a flexbox container!</div>
I'm an inline flexbox container!
<div class="c-d-inline-flex c-p-2 bd-highlight">I'm an inline flexbox container!</div>

Responsive variations also exist for .c-d-flex and .c-d-inline-flex.

  • .c-d-flex
  • .c-d-inline-flex
  • .c-d-sm-flex
  • .c-d-sm-inline-flex
  • .c-d-md-flex
  • .c-d-md-inline-flex
  • .c-d-lg-flex
  • .c-d-lg-inline-flex
  • .c-d-xl-flex
  • .c-d-xl-inline-flex

Direction

Set the direction of flex items in a flex container with direction utilities. In most cases you can omit the horizontal class here as the browser default is row. However, you may encounter situations where you needed to explicitly set this value (like responsive layouts).

Use .c-flex-row to set a horizontal direction (the browser default), or .c-flex-row-reverse to start the horizontal direction from the opposite side.

Fill

Use the .c-flex-fill class on a series of sibling elements to force them into widths equal to their content (or equal widths if their content does not surpass their border-boxes) while taking up all available horizontal space.

Flex item with a lot of content

<div class="c-d-flex bd-highlight">
  <div class="c-p-2 c-flex-fill bd-highlight">Flex item with a lot of content</div>
  <div class="c-p-2 c-flex-fill bd-highlight">Flex item</div>
  <div class="c-p-2 c-flex-fill bd-highlight">Flex item</div>
</div>

Grow and shrink

Use .c-flex-grow-* utilities to toggle a flex item’s ability to grow to fill available space. In the example below, the .c-flex-grow-1 elements uses all available space it can, while allowing the remaining two flex items their necessary space.

Flex item
Flex item
Third flex item
<div class="c-d-flex bd-highlight">
  <div class="c-p-2 c-flex-grow-1 bd-highlight">Flex item</div>
  <div class="c-p-2 bd-highlight">Flex item</div>
  <div class="c-p-2 bd-highlight">Third flex item</div>
</div>

Use .c-flex-shrink-* utilities to toggle a flex item’s ability to shrink if necessary. In the example below, the second flex item with .c-flex-shrink-1 is forced to wrap it’s contents to a new line, “shrinking” to allow more space for the previous flex item with .c-w-100.

Flex item
Flex item
<div class="c-d-flex bd-highlight">
  <div class="c-p-2 c-w-100 bd-highlight">Flex item</div>
  <div class="c-p-2 c-flex-shrink-1 bd-highlight">Flex item</div>
</div>

Responsive variations also exist for c-flex-grow and c-flex-shrink.

Auto margins

Flexbox can do some pretty awesome things when you mix flex alignments with auto margins. Shown below are three examples of controlling flex items via auto margins: default (no auto margin), pushing two items to the right (.c-mr-auto), and pushing two items to the left (.c-ml-auto).

Unfortunately, IE10 and IE11 do not properly support auto margins on flex items whose parent has a non-default c-justify-content value. See this StackOverflow answer for more details.

Order
